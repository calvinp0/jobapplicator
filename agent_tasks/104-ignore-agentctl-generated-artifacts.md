# Task 104: Ignore agentctl Generated Verification Artifacts

## Goal

Prevent `agentctl` from blocking future task runs because of generated verification artifacts such as:

```text
.agentctl/verifications/
```

Current observed failure:

```text
error: git tree is not clean; commit or stash before starting a new run:
?? .agentctl/verifications/
```

This happened after a prior task completed successfully and before the next task worktree was created.

Do not change application behavior.
Do not change frontend behavior.
Do not change backend behavior.

## Background

The harness currently refuses to start a task if the main git tree is dirty.

That is usually correct, but generated harness artifacts should either be:

```text
- cleaned up automatically
- ignored by git
- stored outside the repo
```

The current artifact:

```text
.agentctl/verifications/
```

is generated by the agent workflow and should not block future task runs.

## Inspect

Inspect:

```text
scripts/agentctl.sh
.gitignore
.agentctl/
docs/contracts/agent_orchestration.md
```

Search:

```bash
rg "verifications|git tree is not clean|git status|dirty|clean" scripts .gitignore docs
```

## Required Behavior

Add generated agent artifacts to `.gitignore`:

```gitignore
.agentctl/verifications/
```

Also consider ignoring other generated harness scratch outputs if they exist and are safe to ignore.

Do not ignore:

```text
.agentctl/journal/
```

unless the project already treats journals as generated/untracked.

## Optional Harness Improvement

If practical, update `scripts/agentctl.sh` so preflight dirty-tree checks ignore known generated harness artifacts.

Preferred minimal fix:

```text
.gitignore .agentctl/verifications/
```

Optional stronger fix:

```text
agentctl cleans .agentctl/verifications/ after completion
or stores them outside the repo worktree
```

Do not overcomplicate this task.

## Tests / Verification

Run:

```bash
bash -n scripts/agentctl.sh
git status --short
```

Manual verification:

1. Create the directory:

```bash
mkdir -p .agentctl/verifications/test
touch .agentctl/verifications/test/output.txt
```

2. Confirm:

```bash
git status --short
```

does not show:

```text
?? .agentctl/verifications/
```

3. Run:

```bash
scripts/agentctl.sh work 101-frontend-applications-table-column-consolidation
```

or another ready task.

4. Confirm the run starts instead of failing with a dirty tree caused by `.agentctl/verifications/`.

## Acceptance Criteria

- `.agentctl/verifications/` no longer appears as an untracked dirty file.
- Generated verification artifacts do not block new task runs.
- Existing agent journals are not accidentally ignored unless already intended.
- `agentctl` syntax remains valid.
- A ready task can start after verification artifacts exist.
