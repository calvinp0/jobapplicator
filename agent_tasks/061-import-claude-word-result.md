# Task 061: Import Claude for Word Result

## Goal

Add backend support for importing the finished Claude for Word resume result.

The expected manual/semi-automated flow is:

```text
JobApplicator creates word_handoff/
user opens DOCX in Word
user uses Claude for Word
user saves result to output/word_tailored_resume.docx
JobApplicator imports/validates result
```

Do not change frontend UI in this task unless an existing minimal API test fixture requires it.
Do not implement Gmail.
Do not implement LinkedIn automation.

## Background

Inspect:

```text
backend/app/run_directory.py
backend/app/word_handoff.py
backend/app/main.py
backend/tests/test_word_handoff.py
docs/contracts/claude_run_directory.md
docs/contracts/agent_orchestration.md
```

This task depends on Task 060.

## Required Behavior

Add backend logic to check for and import:

```text
runs/<run_id>/output/word_tailored_resume.docx
```

When present, copy or mark it as:

```text
runs/<run_id>/output/final_resume.docx
```

Also preserve the source file:

```text
runs/<run_id>/output/word_tailored_resume.docx
```

Do not delete auto outputs.

If auto outputs exist, keep them:

```text
output/tailored_resume.md
output/tailored_resume.docx
output/change_log.md
output/claim_audit.md
```

The Word output should be treated as a separate candidate:

```text
output/word_tailored_resume.docx
output/final_resume.docx
```

## Status Behavior

If `word_tailored_resume.docx` exists and is imported successfully:

```text
status = word_result_imported
```

Then, if final validation passes:

```text
status = completed
```

If the Word result does not exist:

```text
status = waiting_for_word_result
```

The API should return a useful message saying where the file should be saved.

## Optional Audit Handling

If Word mode does not produce:

```text
change_log.md
claim_audit.md
```

do not fail solely for that reason in this task, unless current contracts require these for every completed run.

Instead, if missing, create placeholders:

```text
output/change_log.md
output/claim_audit.md
```

The placeholder should clearly say:

```text
Generated by Claude for Word handoff flow.
No structured audit was imported.
Manual review required.
```

Do not pretend an audit was performed if it was not.

## Required Endpoints

Add an endpoint following current route style:

```text
POST /api/runs/{run_id}/import-word-result
```

It should return JSON like:

```json
{
  "run_id": "<run_id>",
  "status": "completed",
  "word_result": "runs/<run_id>/output/word_tailored_resume.docx",
  "final_resume": "runs/<run_id>/output/final_resume.docx",
  "message": "Imported Claude for Word result"
}
```

If the file is missing, return a project-standard response such as:

```json
{
  "run_id": "<run_id>",
  "status": "waiting_for_word_result",
  "expected_output": "runs/<run_id>/output/word_tailored_resume.docx",
  "message": "Save the completed Word document to the expected output path, then import again"
}
```

Use existing status/error conventions where possible.

## Validation

At minimum validate:

```text
- word_tailored_resume.docx exists
- file size > 0
- final_resume.docx exists after import
- final_resume.docx size > 0
```

Do not parse DOCX contents in this task unless existing utilities already do that.

## Run Log

Record:

```text
jobapply: checking for Claude for Word result
jobapply: expected Word result=output/word_tailored_resume.docx
jobapply: imported Word result to output/final_resume.docx
```

If missing:

```text
jobapply: Word result not found yet
```

Do not print secrets.

## Tests

Add/update tests proving:

1. Missing Word result sets status to `waiting_for_word_result`.
2. Missing Word result response includes expected output path.
3. Existing `word_tailored_resume.docx` imports to `final_resume.docx`.
4. Imported final resume has nonzero file size.
5. Existing auto outputs are not deleted.
6. Missing audit files get honest placeholders.
7. Run status becomes `completed` after successful import.
8. Run log records import or waiting state.

Do not require Microsoft Word.
Do not require Claude for Word.
Do not require real Claude Code.

## Acceptance Criteria

- Backend can import completed Claude for Word DOCX.
- Final resume is stored as `output/final_resume.docx`.
- Auto outputs are preserved.
- Missing result puts run into `waiting_for_word_result`.
- Missing audits get honest placeholder files.
- Tests pass.
